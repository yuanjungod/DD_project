# Run step outputs

Per-agent artifacts for a workflow run live on disk next to the session run record. This document matches `agent_service/workflows/agent_outputs.py` and `WorkflowEngine`.

## Path layout

```text
.harness_project/users/{user_id}/workflows/{workflow_template_id}/{engagement_id}/
  sessions/{session_id}/runs/
    {run_id}.json
    outputs/
      {run_id}_outputs/                              # run-level output root
        {run_id}_step_{NNN}_{agent_name}/            # one folder per executed agent step
          README.md                                  # platform handoff (always created if missing)
          …                                          # files the agent writes (json, csv, scripts, etc.)
```

Example:

```text
outputs/run_dd4686272e35_outputs/
  run_dd4686272e35_step_001_tianyancha/
  run_dd4686272e35_step_002_FinancialAnalysisAgent/
```

`NNN` is a zero-padded step sequence for the run (`001`, `002`, …). The folder name is **`{step_id}_{agent_name}`**, not agent name alone.

## Multi-agent workflows

- **Each agent step gets its own subdirectory** under `{run_id}_outputs/`. Outputs are **not** merged into a single shared folder.
- **DAG parallel levels**: agents in the same graph level may run concurrently; each still receives a distinct `step_00N` directory.
- **Same agent twice** in one run (two nodes or replay) → two folders distinguished by step number (`step_001_…`, `step_002_…`).
- **Downstream handoff**: later agents receive `previous_agent_output_folders` and inlined `previous_agent_handoff_readmes` in the prompt (from prior steps’ `output_dir` + `README.md`). They read other files with `view_text_file`.

## Platform vs agent-written files

| File | Who writes it |
| --- | --- |
| `README.md` | Platform creates a placeholder if the agent did not write one (`ensure_step_output_handoff`). Agents are encouraged to overwrite with a real summary. |
| Other files | Agent tools (`execute_shell_command`, `execute_python_code`, etc.) under the assigned `agent_output_dir`. |
| `result.json` | **Optional** — not auto-generated by the platform today. Report fallback may read it when present. |

## API and UI

- `AgentResult.output_dir` and `output_readme_path` on each completed step point at the step folder and its README.
- Backend lists and downloads files under that folder (`backend/app/services/agent_step_output_files.py`).
- Docker execution mode uses container paths under `/workspace/workflow/...` in prompts while host paths remain under `.harness_project/` (see [ADR-0011](adr/0011-workflow-docker-execution.md)).
